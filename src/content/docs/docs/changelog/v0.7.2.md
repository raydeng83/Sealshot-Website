**Released July 22, 2026** ·
[Download](https://github.com/raydeng83/Sealshot-Release/releases/tag/v0.7.2)

## Fixes

- Dragging a single capture from the recent strip to Finder now drops a real,
  full-resolution image file. It previously created a tiny broken alias instead
  of the actual picture.
- The empty area around the image in the editor now behaves like empty canvas:
  clicking it deselects, and you can start a selection box out there and drag
  across the image. Previously clicks out there did nothing at all.
- Dragging a selection box now selects only the objects it fully encloses,
  instead of everything it touches on the way past.
- The Editor/Library/Settings switcher no longer offers an Icon and Text /
  Icon Only / Text Only menu on right-click. Picking one of those could leave
  a stray "View" label sitting under the switcher. (macOS 15 and later.)

## Licensing

- Direct edition: installing a Sealshot version released after your license's
  update window now shows a renewal banner and pauses new captures and
  recordings until you renew or switch back to a covered version. Everything
  already captured stays fully viewable, editable, and exportable.

## Library

- Right-click **All Files** in the Library sidebar to **Export All Files…** —
  bundles your whole library into one package you can AirDrop to another Mac and
  import (optionally passphrase-protected). Works like exporting a collection,
  just for everything.

- Exporting a package now shows how many items are done (e.g. "Packaging 5 of
  10…") instead of a time-left estimate.

## Security

- Your recovery key screen now has "Save Kit…" and "Print…" buttons, so you
  can keep a PDF copy of your recovery code — with instructions — on paper or
  in your own backup, not just copy it to the clipboard.
- Every so often, Sealshot will ask you to confirm you still have your
  recovery code (about once every 3 months). This never happens while your
  library is locked or more than once per launch, and you can always say
  "Remind Me Later" or generate a fresh code on the spot.
- The lock screen now has an "I can't unlock…" option for when you've lost
  both Touch ID access and your recovery code. It explains why Sealshot can't
  bypass encryption for you, then — after you type RESET to confirm — moves
  your encrypted captures to a Locked Archive and turns encryption off so you
  can keep using the app. Nothing is deleted, and archived captures can still
  be restored later if you find your recovery code.
- Captures moved to your Locked Archive now show up as their own section in
  the Library, so you can see what's there instead of it being hidden on
  disk. They're read-only — shown dimmed with a lock badge — since Sealshot
  can't decrypt them without your recovery code, but you can still select
  them and reveal them in Finder.
- Found your recovery code again? You can now restore the Locked Archive: a
  Restore… button in the Library's Locked Archive section (and in Settings →
  Privacy & Security) asks for the code and brings those captures back into
  your library, re-enabling encryption with the recovered key. If you went
  through more than one reset, each round's captures come back with that
  round's own code — items belonging to a different code stay safely in the
  archive, and nothing is ever deleted — even if you'd already set up
  encryption again in the meantime. The restore screen also warns you if
  some internal keys couldn't be brought back, instead of failing silently.
  If restoring an older code needs to set aside your current encryption
  setup to make room, the restore screen now tells you so, and reminds you
  to bring your current setup back with its own recovery code.
- Some Locked Archive items can never come back through Restore… — ones set
  aside when you turned encryption off, rather than from a reset — and
  Sealshot's messaging now says so plainly instead of implying a code will
  restore them: the Library banner drops the Restore… button when nothing in
  the archive is actually restorable, and the restore screen's wording no
  longer promises a code will work for items left behind.
- Successfully entering your recovery code — whether to restore a Locked
  Archive or unlock on a new Mac — now counts as verifying it, the same as
  using the periodic "Verify Your Recovery Code" prompt, so you won't be
  asked again sooner than necessary.
- Fixed the recovery-verify nudge's "Generate New…" button sometimes not
  opening Settings → Privacy & Security when Settings was already open.

## Other

- Live Captures now get keywords and a summary that lists each captured window
  with a one-line description. Scenes taken before this update pick theirs up
  the next time you open them.
- Sealshot now has a 14-day free trial. After it, your existing captures stay
  fully viewable, editable, and exportable — a license is needed only for new
  captures and recordings. Activate by opening your license file (or dropping
  it) in Settings → License. A banner shows when your trial is in its final
  week.
- The Library and Settings tabs use a clean solid backdrop instead of the
  hex-pattern background.
- Sealshot now tells you when capture summaries need Apple Intelligence turned
  on, instead of quietly hiding them — and says plainly when your Mac can't run
  them, so you know nothing is broken.
- Turning Apple Intelligence on in System Settings while a capture is open now
  resumes its summary and keywords automatically as soon as you switch back to
  Sealshot — no more closing and reopening the capture to make it generate.
  Settings → On-Device AI also updates right away instead of showing stale
  status.
